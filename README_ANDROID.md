# دليل بناء تطبيق أندرويد (APK) تلقائياً

لقد قمت بإعداد نظام **أتمتة كامل (Automated Build Pipeline)** لمشروعك باستخدام **GitHub Actions**. إليك كيفية الحصول على التطبيق:

## 1. البناء التلقائي (GitHub Actions)
في كل مرة تقوم فيها بعمل `Push` للكود إلى مستودع GitHub الخاص بك، يقوم النظام بتنفيذ الخطوات التالية تلقائياً:
*   تثبيت جميع الاعتمادات (Dependencies).
*   بناء نسخة الويب (Build Web Assets).
*   مزامنة الملفات مع مشروع الأندرويد (Capacitor Sync).
*   **إنشاء ملف APK جاهز للتثبيت.**

### كيف تجد ملف الـ APK؟
1.  اذهب إلى مستودع المشروع على **GitHub**.
2.  اضغط على تبويب **Actions** في القائمة العلوية.
3.  سوف ترى سير العمل (Workflow) باسم **"Android CI"**.
4.  اضغط على أحدث عملية بناء (Successful build).
5.  في أسفل الصفحة، ستجد قسم **Artifacts**، وبداخله ملف **app-debug** مضغوط يحتوي على الـ APK.

## 2. البناء اليدوي (باستخدام Android Studio)
إذا أردت بناء التطبيق يدوياً على جهازك:
1.  قم بتحميل المشروع (Export to ZIP).
2.  افتح المجلد في جهازك.
3.  شغل الأمر التالي لتحديث الملفات:
    ```bash
    npm run android:sync
    ```
4.  افتح مجلد `android` باستخدام برنامج **Android Studio**.
5.  من القائمة اختر: **Build > Build Bundle(s) / APK(s) > Build APK(s)**.

## 3. مميزات التطبيق المثبت
*   **العمل بدون إنترنت:** يتم تخزين البيانات محلياً والمزامنة لاحقاً.
*   **أداء أسرع:** وصول مباشر لمكونات الهاتف.
*   **تنبيهات:** دعم إرسال إشعارات (Notifications) مستقبلاً.
